= Java gRPC Sample
:toc:
:toclevels: 2

This repository contains three Gradle-based Java 21 modules that demonstrate gRPC communication with Spring Boot 4 and Spring gRPC:

* `product-proto` generates Java classes and gRPC stubs from the shared product contract.
* `products-grpc` exposes product-related gRPC endpoints and keeps product data in memory.
* `orders-grpc` exposes order-related gRPC endpoints and calls `products-grpc` to enrich order items with product names.

== Project Structure

[cols="1,3"]
|===
| Module | Responsibility

| `product-proto`
| Owns `product.proto` and packages the generated Java classes into `product-proto-0.0.1-SNAPSHOT-plain.jar`.

| `products-grpc`
| Spring Boot gRPC server running on port `9091`. Implements product creation and lookup operations.

| `orders-grpc`
| Spring Boot gRPC server running on port `9090`. Implements order operations and uses a gRPC client channel named `product-service` to call `products-grpc`.
|===

== How The Services Work

`products-grpc` starts with three sample products in `ProductRepository`. New products are added to the same in-memory list, so all product data is lost when the application restarts.

`orders-grpc` stores orders in an in-memory list inside `OrderRepository`. When an order is created or fetched, the service calls `products-grpc` with each `productId` and fills `productName` from the product response.

Because both services use in-memory repositories, this project is intended as a learning or demo setup rather than a production-ready system.

== gRPC Contracts

=== Product Contract

Source: `product-proto/src/main/proto/product.proto`

Service: `ProductService`

* `CreateProduct(ProductRequest) returns (ProductResponse)`
* `GetProducts(Empty) returns (ProductResponse)`
* `GetProduct(ProductIdRequest) returns (Product)`

Messages:

* `ProductRequest`: `name`, `price`, `active`
* `Product`: `id`, `name`, `price`, `active`
* `ProductResponse`: repeated `products`
* `ProductIdRequest`: `id`

=== Order Contract

Source: `orders-grpc/src/main/proto/order.proto`

Service: `OrderService`

* `CreateOrder(Order) returns (Order)`
* `GetOrders(Empty) returns (OrderResponse)`

Messages:

* `Order`: `id`, `status`, repeated `items`, `createdBy`
* `OrderItem`: `productId`, `productName`, `price`, `quantity`
* `OrderResponse`: repeated `order`
* `Status`: `NEW`, `SOURCING`, `RELEASED`, `COMPLETED`

== Build And Run

=== Prerequisites

* Java 21
* A terminal with permission to run the included Gradle wrappers

=== 1. Build The Shared Proto Jar

The service modules depend on the generated jar from `product-proto`, so build that module first:

[source,bash]
----
cd product-proto
./gradlew clean build
----

This produces:

`product-proto/build/libs/product-proto-0.0.1-SNAPSHOT-plain.jar`

=== 2. Start The Product Service

[source,bash]
----
cd products-grpc
./gradlew bootRun
----

The gRPC server listens on port `9091`.

=== 3. Start The Order Service

Open another terminal:

[source,bash]
----
cd orders-grpc
./gradlew bootRun
----

The gRPC server listens on port `9090` and connects to `localhost:9091` for product lookups.

== Typical Flow

1. Create one or more products through `ProductService.CreateProduct`, or use one of the three sample products loaded at startup.
2. Call `ProductService.GetProducts` to obtain valid `productId` values.
3. Create an order through `OrderService.CreateOrder` with `items[].productId` set to existing product ids.
4. Call `OrderService.GetOrders` to retrieve orders enriched with `productName`.

== Example Payload Shapes

=== Create Product Request

[source,json]
----
{
  "name": "Keyboard",
  "price": 49.99,
  "active": true
}
----

=== Create Order Request

[source,json]
----
{
  "status": "NEW",
  "createdBy": "demo-user",
  "items": [
    {
      "productId": "<product-id>",
      "price": 49.99,
      "quantity": 2
    }
  ]
}
----

== Implementation Notes

* `products-grpc` and `orders-grpc` are separate Gradle projects, not submodules of a single multi-module build.
* Both services use Spring Boot 4.0.5 and Spring gRPC 1.0.2.
* `orders-grpc` consumes generated product classes by referencing the jar built from `../product-proto/build/libs/...`.
* `orders-grpc` generates its own order-related classes from `order.proto`.
